## 결론 먼저

이미지 글자 추출은 사진, 스크린샷, 스캔 PDF 안의 글자를 복사 가능한 텍스트로 바꾸는 작업입니다.
보통 OCR이라고 부릅니다.

가장 빠른 방법은 온라인 OCR 사이트에 파일을 올리는 것입니다.
하지만 주민등록번호, 계약서, 견적서, 세금계산서, 고객 정보처럼 민감한 내용이 들어 있다면 먼저 멈춰야 합니다.

**파일을 어디론가 업로드해도 되는 문서인지 먼저 판단하고, 민감한 문서라면 브라우저에서 처리되는 OCR 도구나 로컬 프로그램을 우선 검토하는 편이 안전합니다.**

이 글에서는 상황별로 어떤 방법을 쓰면 좋은지, PDF OCR과 이미지 OCR은 무엇이 다른지, 인식률을 높이려면 무엇을 확인해야 하는지 정리합니다.

## 이 글이 필요한 사람

이 글은 이런 상황에 맞습니다.

- 이미지 속 글자를 복사해서 문서나 메모에 붙여넣고 싶다.
- 스캔한 PDF에서 텍스트 선택이 안 된다.
- 사진 글자 추출 사이트를 써도 되는지 개인정보가 걱정된다.
- OCR 결과가 자꾸 깨져서 인식률을 높이고 싶다.
- 설치 없이 브라우저에서 간단히 OCR을 해보고 싶다.
- PDF, 영수증, 스크린샷, 계약서 이미지 중 어떤 방식이 맞는지 헷갈린다.

먼저 핵심은 파일 유형을 구분하는 것입니다.
텍스트가 선택되는 PDF와 스캔 이미지로 된 PDF는 처리 방법이 다릅니다.

## 파일 유형별 빠른 선택표

| 상황 | 먼저 해볼 방법 | 이유 |
| --- | --- | --- |
| PDF에서 텍스트가 마우스로 선택된다 | OCR 말고 텍스트 복사 또는 PDF 텍스트 추출 | 이미 텍스트가 들어 있는 PDF라 OCR보다 정확하다 |
| PDF에서 텍스트 선택이 안 된다 | PDF OCR | 종이를 스캔한 이미지 PDF일 가능성이 높다 |
| 사진 속 안내문, 영수증, 책 페이지를 뽑고 싶다 | 이미지 OCR | 사진 자체를 분석해야 한다 |
| 주민번호, 계좌번호, 계약서가 들어 있다 | 업로드형 OCR보다 브라우저 처리 또는 로컬 프로그램 우선 | 민감한 파일을 외부 서버에 올리는 위험을 줄인다 |
| 표, 영수증, 금액, 날짜를 정리하고 싶다 | OCR 후 사람이 검수 | OCR은 텍스트 추출은 가능하지만 표 구조를 완벽히 복원하지 못할 수 있다 |
| 손글씨를 인식하고 싶다 | 기대치를 낮추고 수동 확인 | 일반 OCR은 인쇄 글자에 더 강하다 |

처음부터 “어떤 OCR 사이트가 제일 좋나”로 접근하면 실수하기 쉽습니다.
먼저 내 파일이 업로드해도 되는 파일인지, 그리고 OCR이 필요한 파일인지부터 확인해야 합니다.

## 이미지 글자 추출과 PDF OCR 차이

이미지 글자 추출은 이미지 파일 안의 글자를 읽는 작업입니다.
예를 들면 JPG, PNG, WebP, 휴대폰 사진, 스크린샷 같은 파일입니다.

PDF OCR은 조금 다릅니다.
PDF는 크게 두 종류가 있습니다.

| PDF 종류 | 특징 | 추천 방법 |
| --- | --- | --- |
| 텍스트 PDF | 마우스로 글자를 드래그할 수 있다 | OCR보다 텍스트 추출이 낫다 |
| 스캔 PDF | 글자가 이미지처럼 박혀 있어 선택이 안 된다 | OCR이 필요하다 |

스캔 PDF는 겉보기에는 문서처럼 보이지만 내부적으로는 종이 문서를 찍은 이미지 묶음에 가깝습니다.
그래서 복사 가능한 텍스트를 얻으려면 각 페이지를 이미지로 읽고 OCR을 돌려야 합니다.

Tesseract.js FAQ에서도 PDF 자체를 직접 인식하는 방식이 아니라, PDF를 이미지로 렌더링한 뒤 OCR을 수행하는 접근이 필요하다고 설명합니다.
이 사이트의 PDF OCR Extractor도 같은 방식에 가깝습니다.
PDF.js로 PDF를 페이지 이미지처럼 처리하고, Tesseract.js로 글자를 인식합니다.

## 온라인 OCR 사이트 쓰기 전 개인정보 체크

OCR 사이트를 쓸 때 가장 먼저 볼 것은 “무료인지”가 아닙니다.
**내 파일이 외부 서버로 업로드되는지**입니다.

아래 항목이 들어 있으면 아무 사이트에나 올리지 않는 편이 좋습니다.

- 주민등록번호, 여권번호, 운전면허번호
- 계좌번호, 카드번호, 결제 내역
- 계약서, 견적서, 세금계산서
- 고객 이름, 전화번호, 이메일, 주소
- 회사 내부 문서, 제안서, 회의록
- 병원 기록, 보험 서류, 법률 문서
- 학생 명단, 성적표, 출석부

업로드형 OCR 사이트가 무조건 위험하다는 뜻은 아닙니다.
다만 파일이 서버로 전송되는 구조라면, 사용 약관과 개인정보 처리방침을 확인해야 합니다.
민감한 문서라면 더 보수적으로 판단해야 합니다.

간단한 기준은 이렇습니다.

| 문서 유형 | 추천 판단 |
| --- | --- |
| 공개된 안내문, 제품 설명서, 책 일부 메모 | 온라인 OCR 사용 가능성이 비교적 높음 |
| 개인 메모, 일반 스크린샷 | 내용 확인 후 사용 |
| 계약서, 신분증, 세금 문서, 고객 정보 | 업로드형 도구는 피하는 쪽 권장 |
| 회사 내부 문서 | 회사 보안 정책 확인 전 업로드 금지 |

OCR을 하기 전에 민감한 영역을 가릴 수 있다면 먼저 가리세요.
예를 들어 주민번호 뒷자리, 전화번호 일부, 계좌번호 일부는 이미지 편집 도구로 지운 뒤 처리하는 편이 낫습니다.

## 브라우저에서 처리하는 OCR이 유리한 경우

브라우저 처리형 OCR은 파일을 서버에 올리는 대신, 사용자의 브라우저 안에서 OCR 작업을 수행하는 방식입니다.
Tesseract.js는 JavaScript와 WebAssembly 기반으로 브라우저 또는 Node.js에서 OCR을 실행할 수 있는 라이브러리입니다.
공식 사이트에서는 100개 이상의 언어와 방향 감지, 단어·문자 위치 정보 등을 지원한다고 설명합니다.

브라우저 처리형 OCR이 유리한 경우는 다음과 같습니다.

- 민감한 파일을 외부 서버에 올리고 싶지 않다.
- 설치 없이 빠르게 글자만 뽑고 싶다.
- 스크린샷이나 간단한 문서 사진을 처리한다.
- 결과를 직접 보고 수정할 수 있다.
- 대량 문서보다 몇 장의 이미지나 PDF를 처리한다.

다만 한계도 있습니다.

- 큰 PDF는 브라우저 메모리와 성능 영향을 받는다.
- 첫 실행 때 OCR 언어 데이터나 라이브러리를 내려받을 수 있다.
- 흐린 사진, 기울어진 사진, 손글씨는 정확도가 낮을 수 있다.
- 표 구조, 레이아웃, 서식까지 완벽히 복원한다고 기대하면 안 된다.
- 중요한 문서는 OCR 결과를 반드시 사람이 검수해야 한다.

즉, 브라우저 OCR은 “항상 최고 정확도”라서 쓰는 것이 아닙니다.
**업로드 부담을 줄이고, 간단한 글자 추출을 빠르게 처리하기 좋은 선택지**입니다.

OCR은 원본 이미지 품질의 영향을 크게 받습니다.

## OCR 인식률 높이는 체크리스트

OCR 결과가 이상하다면 도구를 바꾸기 전에 원본 상태를 먼저 확인하세요.
대부분의 실패는 OCR 엔진보다 이미지 품질에서 시작됩니다.

| 문제 | 해결 방법 |
| --- | --- |
| 글자가 흐리다 | 더 높은 해상도로 다시 스캔하거나 촬영한다 |
| 사진이 기울어졌다 | 문서 가장자리를 맞추고 정면에서 다시 찍는다 |
| 그림자가 있다 | 조명을 고르게 하고 그림자를 피한다 |
| 배경이 복잡하다 | 문서 영역만 잘라낸다 |
| 글자가 너무 작다 | 확대하거나 고해상도 스캔을 사용한다 |
| 한국어와 영어가 섞였다 | OCR 언어를 `한국어 + English`처럼 맞춘다 |
| 표가 깨진다 | OCR 결과를 표 그대로 믿지 말고 CSV나 수동 정리를 병행한다 |
| 손글씨가 많다 | 일반 OCR 정확도가 낮을 수 있으므로 수동 확인을 전제로 한다 |

휴대폰으로 찍는다면 아래 순서가 좋습니다.

1. 문서를 평평한 곳에 둔다.
2. 화면에 문서가 꽉 차게 맞춘다.
3. 그림자가 생기지 않게 한다.
4. 흔들리지 않게 찍는다.
5. 필요 없는 배경을 잘라낸다.
6. OCR 결과를 원본과 대조한다.

특히 영수증이나 계약서처럼 숫자가 중요한 문서는 반드시 다시 확인해야 합니다.
`8`과 `B`, `0`과 `O`, `1`과 `I`처럼 비슷한 문자는 쉽게 헷갈릴 수 있습니다.

## 자주 막히는 부분

### PDF인데 왜 텍스트가 선택되지 않나요?

종이 문서를 스캔해서 만든 PDF일 가능성이 큽니다.
겉보기에는 PDF지만 내부 글자가 실제 텍스트가 아니라 이미지입니다.
이 경우 PDF OCR이 필요합니다.

### OCR 결과가 줄바꿈이 이상합니다

OCR은 이미지 안의 글자 위치를 추정해서 텍스트로 바꿉니다.
문단, 표, 열 구조를 사람이 읽는 것처럼 완벽히 이해하지 못할 수 있습니다.
복사 후 메모장이나 문서 편집기에서 줄바꿈을 정리해야 할 때가 많습니다.

### 온라인 OCR 사이트에 올리면 파일이 저장되나요?

사이트마다 다릅니다.
어떤 서비스는 서버에서 처리하고, 어떤 도구는 브라우저에서 처리합니다.
개인정보가 들어 있다면 업로드 전에 개인정보 처리방침과 처리 방식을 확인해야 합니다.
확실하지 않으면 민감한 파일은 올리지 않는 편이 안전합니다.

### 브라우저 OCR이면 무조건 파일이 밖으로 안 나가나요?

도구 구현 방식에 따라 다릅니다.
브라우저에서 OCR을 처리하더라도 라이브러리나 언어 데이터를 CDN에서 내려받을 수 있습니다.
또한 사이트가 별도 업로드 기능을 함께 쓸 수도 있습니다.
따라서 “브라우저 처리”라고 쓰여 있어도 실제 안내와 네트워크 동작을 확인하는 편이 좋습니다.

### 손글씨도 잘 되나요?

일반적인 Tesseract 계열 OCR은 인쇄 글자에 더 강합니다.
Tesseract.js FAQ도 손글씨 인식은 모델의 가정과 맞지 않아 결과가 좋지 않을 수 있다고 설명합니다.
손글씨 문서는 OCR 결과를 보조 자료로만 보고, 사람이 다시 확인해야 합니다.

## 언제 어떤 방법을 고르면 좋을까?

마지막으로 정리하면 이렇습니다.

| 목적 | 추천 |
| --- | --- |
| 빠르게 공개 문서 글자만 복사 | 온라인 OCR 또는 브라우저 OCR |
| 개인정보가 있는 문서 처리 | 브라우저 OCR 또는 로컬 프로그램 우선 |
| 스캔 PDF를 검색 가능한 PDF로 만들기 | PDF OCR 전용 도구 |
| 이미지 한두 장에서 텍스트 추출 | 브라우저 OCR |
| 많은 문서를 반복 처리 | 데스크톱 OCR 프로그램 또는 자동화 도구 검토 |
| 정확도가 중요한 계약서·숫자 문서 | OCR 후 반드시 수동 검수 |

이미지 글자 추출은 이제 어렵지 않습니다.
문제는 “어떤 도구가 제일 빠른가”보다 “내 파일을 어디까지 맡겨도 되는가”입니다.

공개 자료라면 온라인 도구가 편할 수 있습니다.
민감한 문서라면 업로드 전에 한 번 더 확인하고, 가능하면 브라우저 처리형 OCR이나 로컬 도구를 먼저 검토하세요.
